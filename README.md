# What Are SOLID Principles?
<h3>SOLID is a set of 5 design principles that help developers write clean and maintainable code:</h3>
<ol>
<li>S - Single Responsibility Principle (SRP): class should have only one reason to change, only responsible for one thing </li>
<li>O - Open/Closed Principle (OCP): Open for extension, closed for modification </li>
<li>L - Liskov Substitution Principle (LSP): subclass do not override parent method</li>
<li>I - Interface Segregation Principle (ISP): Do not force classes to implement unused methods</li>
<li>D - Dependency Inversion Principle (DIP): higher level module do not depend on low level module, use abstraction</li>
</ol>
